1. Жобаны басқару
1.1 Әдістемені таңдау

Жобаны іске асыру үшін Scrum әдістемесі таңдалды.
Бұл әдіс жұмысты кезең-кезеңімен ұйымдастыруға және нәтижені уақытында алуға мүмкіндік береді.

Негізгі элементтері:

Өнім бэклогы (Jira немесе GitHub Projects)

2 апталық спринттер

Күнделікті жиналыс (Daily – 15 минут)

Әр спринт соңында нәтижені талдау (Retrospective)

1.2 Команда құрамы және міндеттері

Project Manager / Scrum Master – жобаны жоспарлау, мерзімді бақылау

System Analyst – талаптарды анықтау, жүйелік сипаттама жасау

Frontend Developer – пайдаланушы интерфейсін әзірлеу

Backend Developer – серверлік логиканы жүзеге асыру

DevOps Engineer – сервер мен CI/CD жүйесін баптау

1.3 Жобаны іске асыру кезеңдері

1-кезең (1-ай):

Жобаның архитектурасын жобалау

Технологиялық стек таңдау

Мәліметтер базасының құрылымын жасау

2-кезең (2-ай):

MVP нұсқасын әзірлеу

Негізгі функционалды іске асыру

Алғашқы тестілеу жүргізу

3-кезең (3-ай):

Жүйені толық тестілеу

Қателерді түзету

1.0 нұсқаны іске қосу

2. Бағдарламалық инфрақұрылым
2.1 Қолданылатын технологиялар

GitHub / GitLab – нұсқаларды басқару

Docker – қосымшаны контейнерлеу

Kubernetes – жүйені масштабтау

2.2 CI/CD жүйесі

Кодты репозиторийге жүктеу

Автоматты тестілеу

Қауіпсіздік тексерісі

Docker бейнесін жасау

Серверге автоматты түрде орналастыру

2.3 Инфрақұрылым код ретінде

Серверлер мен сервистер Terraform арқылы орнатылады

Инфрақұрылым бір командамен іске қосылады

3. Практикалық бөлім

Архитектуралық схема дайындау

Dev, Stage, Prod орталарын бөлу

Мониторинг жүйесін орнату

Қауіпсіздік параметрлерін баптау

Қорытынды

Бұл жоспар жобаны уақытында және сапалы орындауға мүмкіндік береді.
Жоба нәтижесінде толық жұмыс істейтін жүйе алынады.
